# ニホンゴマスター (Nihongo Master)

AIによる学習進捗分析機能付きの日本語学習アプリ。JLPT N5〜N1を網羅し、1アカウントで学習者プロフィールを最大20人まで作成できるサブスクリプション型サービスのフルスタック実装（プロトタイプ）です。

## 主な機能

- **N5〜N1の学習コンテンツ**: 語彙・文法・聴解（文字起こし形式）・読解のクイズをレベル別に収録（現状はサンプルデータ）。
- **AI進捗分析** (`src/lib/ai.ts`):
  - 弱点・得意分野の可視化（カテゴリー別正答率）
  - 学習ペース・合格可能性の予測
  - 次に取り組むべき学習内容のレコメンド
  - `ANTHROPIC_API_KEY` を設定すると、分析結果をもとにClaudeが自然な日本語の講評を生成します（未設定でもルールベースの講評にフォールバックするため、常に動作します）。
- **1アカウント最大20学習者**: `src/lib/repo/learners.ts` の `createLearner()` がアカウントの `max_learners`（デフォルト20）を超えないよう強制します。Netflixのプロフィール選択のようなUIで学習者を切り替えられます。
- **サブスクリプション課金の土台**: Stripe Checkout / Webhook のルートを実装済み（`STRIPE_SECRET_KEY` 等を設定すると本番同様に動作。未設定時は分かりやすいメッセージを返し、アプリの他機能はそのまま使えます）。
- **認証**: メール+パスワード、bcryptハッシュ、JWTのhttpOnly Cookieセッション。

## セットアップ

```bash
npm install
cp .env.example .env.local   # JWT_SECRET だけは必ず変更してください
npm run dev
```

`http://localhost:3000` を開き、新規登録から始めてください。データは `data/app.sqlite` に保存されます（初回起動時に自動作成・サンプルコンテンツを自動投入します）。

### 環境変数

`.env.example` を参照してください。`JWT_SECRET` 以外はすべて任意です（未設定でもアプリ全体が動作するように設計されています）。

## デプロイ（Railway推奨）

このアプリはデータを `data/app.sqlite`（sql.js）にファイルとして保存するため、**VercelやNetlifyのようなサーバーレス環境では書き込みが永続化されません**（関数実行のたびにディスクがリセットされる可能性があるため）。デモ用途なら問題ありませんが、実際に会員登録や進捗を保存し続けたい場合は、常時起動するコンテナ＋永続ボリュームを持つホスティングを使ってください。

**Railway** は無料/トライアル枠でも小容量（0.5GB）の永続ボリュームが使えるため、このアプリの規模には十分です（Renderは無料枠に永続ディスクが無く、有料プランが必要です）。

1. GitHubにこのリポジトリをプッシュ（またはリポジトリ作成時にフォルダをアップロード）
2. [railway.app](https://railway.app) にGitHubアカウントでログイン → 「New Project」→「Deploy from GitHub repo」→ このリポジトリを選択
3. サービスの設定から「Volume」を追加し、マウントパスを決める（例: `/data`）
4. 環境変数を設定: `JWT_SECRET`（必須）、`DATA_DIR=/data`（手順3で決めたパスと合わせる）、必要に応じて `ANTHROPIC_API_KEY` や `STRIPE_*`
5. デプロイ完了後、「Settings → Networking → Generate Domain」で公開URLを発行

Next.jsは`next start`が`PORT`環境変数を自動で参照するため、Railway側の設定は不要です。

## アーキテクチャ

- **Next.js 16 (App Router, TypeScript)** — フロントエンドとAPI（Route Handlers）を1つのコードベースで実装。
- **データ層**: このサンドボックス環境ではネイティブバイナリやPrismaのエンジンをダウンロードできなかったため、[`sql.js`](https://github.com/sql-js/sql.js)（純粋なWASM版SQLite、ネイティブ依存なし）を採用しています。`src/lib/db.ts` の `run` / `all` / `get` の3関数だけがSQLに触れる場所で、`src/lib/repo/*.ts` はその上に薄いリポジトリ層を構成しています。
- **本番移行**: 実運用では `prisma/schema.prisma`（同梱、Postgres向けの1:1スキーマ）に切り替えることを想定しています。`src/lib/db.ts` を Prisma Client に差し替えるだけで、APIルートやビジネスロジック（`src/lib/ai.ts` 等）は変更不要です。手順はスキーマファイル冒頭のコメントに記載しています。
- **AI分析エンジン** (`src/lib/ai.ts`): 数値分析（正答率・ペース予測）は常にルールベースで決定的に計算し、それをもとにチャートを描画します。文章講評だけを任意でClaude APIに委譲する設計です。
- **チャート**: `src/components/charts/` にカスタムSVGチャートを実装。配色は dataviz スキルの検証済みパレット（`palette.md`）をそのまま採用し、色覚多様性（CVD）とコントラストを検証済みです。

## 制限事項（このプロトタイプの範囲）

- **コンテンツ量**: 各レベルに語彙8語・文法5パターン・クイズ12問程度のサンプルのみを収録しています。実際のJLPT試験対策には数千語・数百文型規模のコンテンツと、聴解用の音声素材が必要です。
- **聴解問題**: 音声ファイルではなく、会話の文字起こしを読んで答える形式で仮実装しています。
- **決済**: Stripeキー未設定時はCheckoutが「未設定です」を返すのみで、実際の課金は発生しません。
- **データベース**: 開発・デモ用にファイルベースのSQLite（sql.js）を使用しています。複数サーバーでのスケールや同時書き込みが多い本番運用にはPostgres等への移行を推奨します（上記参照）。
- **学習者プロフィールにログイン機能はなし**: 現状はアカウント所有者のみがログインし、学習者プロフィールは家族共有のプロフィール切り替えUI（Netflix方式）です。個別ログインが必要な場合は学習者ごとのPIN等を追加してください。

## ディレクトリ構成

```
src/
  app/                 App Router のページ・APIルート
  components/          クライアントコンポーネント（フォーム・チャート等）
  lib/
    db.ts              sql.js ラッパー（唯一SQLに触れる場所）
    schema.sql          SQLiteスキーマ
    seed.ts / content/  サンプルコンテンツ（N5〜N1）
    auth.ts             JWT / パスワードハッシュ / セッションCookie
    ai.ts               AI進捗分析エンジン
    repo/               リポジトリ層（accounts, learners, content, quiz, progress）
prisma/schema.prisma    本番Postgres移行用のスキーマ（ブループリント、未接続）
```

## 動作確認済みのフロー

このプロトタイプは以下のエンドツーエンドのフローで実際に動作確認を行っています（`npm run build` → `npm run start` → curlでの一連の操作）:

1. 新規登録 → ログインセッション（JWT Cookie）発行
2. 学習者プロフィールを20人まで作成 → 21人目はプランの上限で拒否（403）
3. レベル・カテゴリーを指定してクイズを取得 → 解答を送信 → 正誤判定と解説を返却
4. 複数カテゴリーでクイズを解いたのち、AI進捗分析エンドポイントが弱点（正答率が低いカテゴリー）を正しく検出し、学習ペース・合格可能性・レコメンドを生成
5. 未認証状態での `/dashboard` アクセスは `/login` にリダイレクト
